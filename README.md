# govisit-monitor — мониторинг очередей МВД Израиля (iPhone → Mac → Chrome)

Связка, которая сама подставляет SMS-код и следит, когда на
[govisit.gov.il](https://govisit.gov.il) появится свободный слот в нужном вам
отделении на подходящую дату, и пингует об этом в Telegram.

**Бронируете вы сами** — скрипт только логинит и уведомляет.

```
 iPhone ──(переадресация SMS)──▶ Messages на Mac ──▶ chat.db
                                                       │
                                       otp-bridge.py читает код
                                                       │  http://127.0.0.1:8765
                                                       ▼
   ваш обычный Chrome  ◀── userscript забирает код, подставляет, жмёт «дальше»,
                           следит за /location ──▶ Telegram при совпадении
```

Почему именно так: iOS не даёт сторонним приложениям читать SMS, поэтому код
ловим на маке из базы Messages. А автоматизацию держим в **вашем реальном
Chrome** — он наследует уже пройденную защиту Akamai, в отличие от «чистого»
автобраузера.

---

## Состав

| Файл | Где работает | Что делает |
|------|--------------|-----------|
| `otp-bridge.py` | Mac (терминал) | читает SMS-код из chat.db, отдаёт на `127.0.0.1:8765` |
| `govisit-monitor.user.js` | Chrome (Tampermonkey) | логин по OTP, мониторинг отделений, Telegram |
| `com.govisit.otpbridge.plist` | Mac (launchd) | автозапуск моста при входе в систему |

---

## Шаг 1. Переадресация SMS на Mac

На iPhone: **Settings → Messages → Text Message Forwarding** → включить для
вашего Mac. Проверьте: отправьте себе SMS — она должна появиться в приложении
**Messages** на маке. (Оба устройства — один Apple ID, iMessage включён.)

## Шаг 2. Full Disk Access терминалу

macOS не пустит к `chat.db` без этого:
**System Settings → Privacy & Security → Full Disk Access** → добавить
**Terminal** (или iTerm). После добавления — перезапустить терминал.

## Шаг 3. Запуск моста

```bash
cd govisit-monitor
# 1) откройте otp-bridge.py и поменяйте TOKEN на свою длинную случайную строку
python3 otp-bridge.py
```

Проверка, что мост видит базу и коды:

```bash
curl 'http://127.0.0.1:8765/health'
# {"ok": true, "db_exists": true}

# отправьте себе SMS с числом (для этого теста временно поставьте SENDER_CONTAINS = [] — SMS от самого себя дефолтный фильтр отсекает), потом:
curl 'http://127.0.0.1:8765/otp?token=ВАШ_ТОКЕН'
# {"code":"123456","age_seconds":3.1,"fresh":true,...}
```

Если `code:null` — смотрите раздел «Если код не извлекается».

Извлечение кода уже настроено под govisit: SMS от отправителя `GoVisit`, код
ищется после ивритской фразы «קוד האימות» (посторонние цифры — например «5» из
«תקף ל-5 דקות» — не мешают). Логику можно проверить без SMS и без chat.db:

```bash
python3 -m unittest test_otp_extraction -v
```

## Шаг 4. Userscript в Chrome

1. Поставьте расширение **Tampermonkey** в ваш обычный Chrome.
2. Tampermonkey → **Create a new script** → вставьте `govisit-monitor.user.js`.
3. В блоке `CONFIG` впишите:
   - `BRIDGE_TOKEN` — тот же, что в `otp-bridge.py`;
   - `PHONE`, `TEUDAT_ZEUT` — по желанию (для автозаполнения);
   - `DESIRED_OFFICES` — подстроки названий нужных отделений на иврите
     (напр. `["ירושלים"]`);
   - `DESIRED_DATE_BEFORE` — крайняя дата, напр. `"2026-09-01"` (пусто = любая);
   - `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` — для уведомлений (см. ниже).
4. Сохраните. Зайдите на govisit.gov.il — справа снизу появится статус-панель.

> ⚠️ Не коммитьте свой `TEUDAT_ZEUT`, телефон и токены в git. Держите их только
> в локальной копии скрипта в Tampermonkey.

## Шаг 5. Telegram-бот (уведомления)

1. Напишите [@BotFather](https://t.me/BotFather) → `/newbot` → получите
   `TELEGRAM_BOT_TOKEN`.
2. Напишите своему боту любое сообщение.
3. Узнайте `TELEGRAM_CHAT_ID`:
   ```bash
   curl 'https://api.telegram.org/botВАШ_ТОКЕН/getUpdates'
   ```
   возьмите `result[].message.chat.id`.

## Калибровка (1 раз, ~2 минуты)

Скрипт авто-определяет поля и ответ API эвристиками, но точные детали лучше
подтвердить:

1. Откройте DevTools (`Cmd+Opt+I`) → вкладка **Network**.
2. Пройдите путь до страницы отделений вручную один раз.
3. Найдите XHR/fetch-ответ со списком отделений (JSON). Запомните кусок его URL
   и впишите в `CONFIG.PARSE.apiUrlContains` (по умолчанию там уже
   `location/appointment/branch/slot/availab`).
4. На странице кода: если поле OTP не подставляется — скопируйте его CSS-селектор
   (правый клик по полю → Copy → Copy selector) в `CONFIG.SELECTORS.otpInput`.

В консоли скрипт печатает каждый перехваченный ответ и разобранные отделения —
по ним видно, что подкрутить.

## Автозапуск моста (launchd)

Чтобы мост поднимался сам при входе в систему:

```bash
# поправьте путь к otp-bridge.py внутри plist, затем:
cp com.govisit.otpbridge.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.govisit.otpbridge.plist
# остановить: launchctl unload ~/Library/LaunchAgents/com.govisit.otpbridge.plist
```

## Если код не извлекается

- `db_exists:false` → не включена переадресация SMS или нет Full Disk Access.
- `code:null`, хотя SMS пришла:
  - по умолчанию код берётся только из SMS, чей отправитель содержит `govisit`
    (в Messages это `GoVisit`). Если у вас отправитель выглядит иначе (короткий
    номер и т.п.) — поправьте `SENDER_CONTAINS` в `otp-bridge.py`. Мост логирует
    отправителя только для прошедших фильтр SMS: чтобы увидеть его, временно
    поставьте `SENDER_CONTAINS = []`, перезапустите мост и запросите новый
    код — SMS старше `CODE_TTL` в лог не попадут;
  - код ищется после ивритской фразы «קוד האימות»; если govisit сменил текст —
    сработает fallback `CODE_REGEX` (отдельно стоящее число 4–8 цифр), но лучше
    подправить `ANCHORED_CODE_REGEX` под новый текст;
  - на новых macOS текст лежит в `attributedBody` — скрипт это умеет;
  - проверьте `CODE_TTL` — старые коды не отдаются специально (сейчас 300 с,
    как заявленные в SMS «5 минут»).
- Взялось не то число → чаще всего это fallback `CODE_REGEX` на SMS без якоря:
  добавьте `TEXT_CONTAINS` с ключевым словом из сообщения или сузьте
  `CODE_REGEX`; если неправильное число идёт сразу после якорной фразы —
  сузьте `ANCHORED_CODE_REGEX` (например, до точной длины кода).

## Этика и безопасность

- Всё крутится на **вашем** аккаунте, **вашем** маке и **ваших** SMS.
- Интервал опроса намеренно щадящий (минуты, со случайным джиттером). **Не
  снижайте его до секунд** — иначе Akamai забанит, и станет только хуже.
- Сессия живёт по cookie: после логина повторный SMS нужен редко (только когда
  сессия протухнет), так что мост дёргается нечасто.
- Мост слушает только `127.0.0.1` и требует токен — наружу коды не утекают.
- Это инструмент мониторинга для личной записи, а не обхода очереди: бронируете
  вы сами, руками.

## Ограничения

- Mac-часть и работа против живого сайта проверяются только у вас — здесь, в
  облаке, их запустить нельзя. Логика извлечения OTP покрыта тестами в репо
  (`test_otp_extraction.py`: text + attributedBody + якорная регулярка govisit);
  TTL и фильтр отправителя тестами не покрыты. Разбор списка отделений
  проверялся локально.
- Точные селекторы форм и URL API govisit могут отличаться — под это и сделан
  раздел «Калибровка». Скрипт спроектирован так, чтобы правка была в 1–2 строки.
